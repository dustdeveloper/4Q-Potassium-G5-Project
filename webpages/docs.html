(barebones) Game Documentation

This document provides overview on the controls of the game and the game's scenarios.

Controller Overview

There are only 4 keys that help in controlling the system, being:

  • Q : Open the COMMAND modal
  • W : Select ALL UNITS
  • O : Select ONE UNIT (current unit selected)
  • P : Send COMMAND

Possible Scenarios

Two values affect the system: coolant_in_flow and coolant_out_flow. When combined, they make _coolant_amount_rate that gets added to the main coolant_amount.
Positive values add to the amount, and vice versa for the Negative values.

When handling the system, it is advised that you must carefully read the REF VALUE located in any ACTIVE LOG.

EXAMPLE: "Warning detected at UNIT 05 [REF :: AAA_45655]" "Signal detected at UNIT 01 [REF :: DEAD]"

The current scenarios are possible with the system:

  • DISCONNECTED : This unit is recognized to be DISCONNECTED. Nothing can be done to restore it.
  • AAA_0001: Chamber pressure is above the safety threshold (chamber_pressure > 7)
    • Attempt to LOWER the coolant_amount by making lower base.
  • AAA_0002: Power input is under the safety threshold (power_in < 400)
    • Attempt to RESTORE the coolant_amount by making higher base (Not higher than 1).
  • AAA_0003: Unit state changed to "chamber_damage". Cannot be reverted.
